概述

SpringCloud Gateway是基于Spring5、SpringBoot2和ProjectReactor等技术开发的服务网关，旨在提供一种简单、有效的API路由管理功能。

在本篇中，我们将以 SpringCloudAlibaba 入门笔记（五） 为基础，来整合SpringCloud Gateway服务网关，并通过Sentinel实现网关限流。

在上面的配置中，我们可以看到gateway的route由四个部分组成：

  • id：路由的ID
  • url：匹配路由的转发地址
  • predicates：谓语、断言，路由转发的判断条件，支持多种方式，如：Path、Before、After、Between、Cookie、Header、Host、Method、Query、RemoteAddr等。
  • filters：过滤器，路由转发请求时所经过的过滤逻辑，可用于修改请求、响应内容，还包括Hystrix过滤器等

经过上述的配置，我们就可以访问服务网关来转发请求了。

项目启动后，gateway会自动注册到nacos中，当我们访问localhost:8006/v1/roles时，gateway就会为我们自动转发到dubbo-provider（localhost:8004）的/roles路径下。

gateway + sentinel网关限流

从 1.6.0 版本开始，Sentinel 提供了 SpringCloud Gateway 的适配模块，可以提供两种资源维度的限流：

  • route 维度：即在 Spring 配置文件中配置的路由条目，资源名为对应的 routeId
  • 自定义 API 维度：用户可以利用 Sentinel 提供的 API 来自定义一些 API 分组

本篇中，我们以route维度的网关限流为例来进行实验。

我们可以在浏览器疯狂访问两个接口，就会发现user_route很容易就会出现流量控制的情况，而role_route基本不会出现，这说明我们的配置起了作用，当然也说明，在1秒内手动连续刷新20次还是很有难度的。

最后，附上一些Gateway断言和过滤器的说明来作为SpringCloudAlibaba的收尾吧，如下：

predicates

Predicate Example Desc
Path Path=/users/** 请求路径满足Path的匹配规则
Before Before=2019-05-01T00:00:00+08:00[Asia/Shanghai] 在某个时间点之前
After After=2019-04-29T00:00:00+08:00[Asia/Shanghai] 在某个时间点之后
Between Between=2019-04-29T00:00:00+08:00[Asia/Shanghai], 2019-05-01T00:00:00+08:00[Asia/Shanghai] 在某个时间段中
Cookie Cookie=hengboy, yuqiyu 请求中所携带Cookie与断言匹配
Header Header=X-Request-Id, \d+ 请求的Header所携带信息与断言匹配
Host Host=**.yuqiyu.com 对指定的主机名转发
Method Method=POST 根据RequestMethod转发
RemoteAddr RemoteAddr=192.168.1.56/24 指定IP访问时转发，24为子网掩码
Query Query=xxx 请求中包含xxx参数则转发
Query Query=xxx, zzz 请求中包含xxx参数，且取值为zzz则转发

filters

Filter Example Desc
StripPrefix Path=/v1/roles/**
StripPrefix=1		 去掉第一部分URL路径，变为 /roles/**
PrefixPath Path=/hello/**
PrefixPath=/path		 增加部分前缀， 变为 /path/hello/**
